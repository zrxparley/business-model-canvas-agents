# 完整诊断工作流

## 适用场景

- 新业务上线前的全面体检
- 现有业务季度复盘
- 融资前的尽职调查自检
- 转型期的模式重构

---

## 工作流 6 阶段

```
┌─────────────────────────────────────────┐
│  Phase 1 · 需求解读（总指挥）            │
│  → 老板输入业务背景，转化为研究指令      │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  Phase 2 · 垂直深挖（9 个模块专家并行）   │
│  → 每个专家深度分析自己模块，输出标准卡片 │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  Phase 3 · 横向穿透（3 个可行性专家并行） │
│  → 基于 9 张卡片，输出 3 份可行性报告    │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  Phase 4 · 咬合度检查（逻辑审查官）       │
│  → 检查模块一致性 + 三份可行性报告内部逻辑│
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  Phase 5 · 视觉翻译（画布专家）           │
│  → 输出 5 张 mermaid 可视化图            │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  Phase 6 · 综合输出（总指挥）            │
│  → 7 件套最终交付物                      │
└─────────────────────────────────────────┘
```

---

## 各阶段详细动作

### Phase 1 · 需求解读（总指挥）
**输入**：老板填写的任务卡（5W1H）
**动作**：
1. 用 5W1H 框架明确问题边界
2. 识别 9 个模块各自的核心追问
3. 识别 3 个可行性的核心穿透点
4. 输出 12 份研究指令（9 模块 + 3 横向）

**输出**：研究指令清单

---

### Phase 2 · 垂直深挖（9 模块专家并行）
**输入**：研究指令清单
**动作**：每个专家独立完成自己模块的 5 步分析
**输出**：9 份模块分析卡片

**并行要求**：9 个专家之间无依赖，可完全并行

---

### Phase 3 · 横向穿透（3 可行性专家并行）
**输入**：9 份模块分析卡片
**动作**：每个可行性专家基于 9 张卡片做横向分析
**输出**：3 份可行性报告

**并行要求**：3 个可行性专家之间无依赖，可完全并行

---

### Phase 4 · 咬合度检查（逻辑审查官）
**输入**：9 份模块卡片 + 3 份可行性报告
**动作**：
1. 一致性矩阵评分
2. Top 5 关键漏洞识别
3. 风险地图绘制
4. 改版方案建议

**输出**：健康度报告

---

### Phase 5 · 视觉翻译（画布专家）
**输入**：所有分析产出
**动作**：
1. 信息解构（每张图只装一个核心结论）
2. 图型选择
3. mermaid 编码
4. 质量自检

**输出**：
- BMC 9 宫格画布
- 模块咬合关系图
- 增长飞轮图
- 客户旅程图
- 优化路径时间线

---

### Phase 6 · 综合输出（总指挥）
**输入**：所有产出
**动作**：
1. 整合 7 件套
2. 评分定级（A/B/C/D）
3. 提炼 Top 5 核心发现 + Top 5 关键风险
4. 给出 P0/P1/P2 优先级建议
5. 决策建议（继续/调整/暂停/终止）

**输出**：最终诊断书

---

## 角色协作矩阵

| 角色 | 与谁对话 | 输入 | 输出 |
|------|---------|------|------|
| 总指挥 | 老板、所有角色 | 业务背景 | 研究指令 + 最终诊断书 |
| 市场可行性 | KP, CH, VP | 模块卡片 | 市场可行性报告 |
| 交付可行性 | CST, KR, KA, RS, CH | 模块卡片 | 交付可行性报告 |
| 客户需求 | CS, VP, CR, CH | 模块卡片 | 客户需求洞察报告 |
| 9 模块专家 | 仅总指挥 | 研究指令 | 模块分析卡片 |
| 逻辑审查官 | 所有分析角色 | 所有报告 | 健康度报告 |
| 画布专家 | 总指挥 | 核心结论 | mermaid 可视化 |

## 节奏建议

- **每月 1 次** · 方式 3（拍砖新想法）—— 30-60 分钟
- **每季度 1 次** · 完整诊断 —— 1-2 个工作日
- **遇到具体问题** · 方式 2（专项突破）—— 2-4 小时
