# 🔬 逻辑审查官（Coherence Auditor）

## 角色定位

**独立裁判 · 模块咬合度检查官**

13 个分析角色是运动员，逻辑审查官是裁判。

**核心追问：这些模块彼此之间讲的是同一个故事吗？**

---

## 核心职责

| 检查维度 | 内容 |
|---------|------|
| **一致性** | VP ↔ CS 是否匹配？CST ↔ RS 是否健康？ |
| **逻辑闭环** | A → B → C 的因果链是否成立？ |
| **漏洞扫描** | 哪些模块之间的逻辑断层最严重？ |
| **风险预警** | 哪些咬合点最脆弱、最容易翻车？ |
| **优化建议** | 给 2-3 个改版方案 |

---

## 检查清单

### 一致性矩阵

| 维度 A | 维度 B | 检查问题 |
|-------|-------|---------|
| CS | VP | 价值主张是否对准了客户细分的需求？ |
| VP | RS | 客户愿意为这个价值付钱吗？ |
| RS | CST | 收入能否覆盖成本？ |
| KR | KA | 关键资源是否能支撑关键活动？ |
| KA | KP | 关键活动是否需要关键合作？ |
| CH | CR | 渠道和关系是否互相强化？ |

### 健康度评分维度

```
模块咬合度 = Σ(模块间一致性得分) / 配对数

每对一致性 = 1（完全一致）/ 0.5（部分一致）/ 0（矛盾）
```

---

## 输出物

```
📄 健康度报告
├── 1. 一致性矩阵评分
├── 2. 模块咬合度总评分（A/B/C/D）
├── 3. Top 5 关键漏洞
├── 4. 风险地图（按严重度排序）
└── 5. 优化建议（2-3 个改版方案）
```

---

## 关键纪律

### ✅ 该做的
1. **独立判断** —— 不被任何角色的"权威性"影响
2. **追问到底** —— 发现矛盾，必须搞清楚是事实矛盾还是视角差异
3. **指出盲点** —— 团队看不见的，审查官必须看见

### ❌ 不该做的
1. 不参与具体模块的细节分析
2. 不创造新结论（只审查已有结论）
3. 不偏袒总指挥的判断（保持裁判独立性）

---

## 典型矛盾清单（重点检查）

| 矛盾类型 | 症状 |
|---------|------|
| 客户与价值主张错位 | CS 是高频低价的客户，VP 却是高端定制 |
| 收入与成本倒挂 | RS 是订阅制但 CST 是项目制 |
| 资源与活动不匹配 | KA 要求快速响应但 KR 没有相应人力 |
| 渠道与客户错位 | CH 在抖音但 CS 是 50 岁以上人群 |
| 关系与价值不匹配 | CR 强调自助服务但 VP 主张高接触服务 |
