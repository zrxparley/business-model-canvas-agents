# 触发关键词命令手册

> **老板只记一个总入口 `/bmc`，用关键词组合精准召唤。**
>
> 📖 **English version**: [commands.md](./commands.md)

---

## 设计原则

```
[动作] + [对象] + [业务]
```

例如：`/bmc 完整诊断 AI 简历优化`

---

## 一、统一入口

### `/bmc`
所有 BMC 天团相关调用都用这个前缀。

---

## 二、核心命令（7 个）

### 1️⃣ `/bmc 完整诊断`

**触发场景**：新业务上线前、季度复盘、融资前自检

**模板**：
```
/bmc 完整诊断
业务：[名称]
客户：[谁]
价值主张：[一句话]
阶段：[想法/MVP/增长/转型]
困惑：[Top 1-3 困惑]
背景：[团队、营收、关键合作]
输出：[完整 7 件套 / 关键模块 / 重点优化]
```

**默认召唤**：14 角色全员

**预计时间**：1-2 小时

**输出**：7 件套诊断书

---

### 2️⃣ `/bmc 专项诊断`

**触发场景**：具体问题快速突破

**模板**：
```
/bmc 专项诊断 [模块名]
业务：[名称]
问题：[具体问题]
```

**示例**：
```
/bmc 专项诊断 CH
业务：AI 简历优化
问题：CAC 涨到 200 元，知乎渠道是不是该换？
```

**默认召唤**：目标模块专家 + 上下游 2-3 个相关专家

**预计时间**：20-30 分钟

---

### 3️⃣ `/bmc 拍砖`

**触发场景**：新想法快速验证

**模板**：
```
/bmc 拍砖
想法：[描述]
目标客户：[谁]
市场：[哪里]
```

**默认召唤**：3 个可行性专家（不召唤 9 模块专家）

**预计时间**：10-15 分钟

**输出**：可行性三角评分 + 一句话拍砖结论

---

### 4️⃣ `/bmc 客户需求`

**触发场景**：想搞清楚"客户到底为什么要买"

**模板**：
```
/bmc 客户需求
业务：[名称]
当前 VP：[现有价值主张]
```

**默认召唤**：客户需求分析专家 + CS + VP

**输出**：JTBD 任务清单 + 客户旅程地图 + VP 匹配度评分

---

### 5️⃣ `/bmc 财务体检`

**触发场景**：现金流、单位经济、增长飞轮相关问题

**模板**：
```
/bmc 财务体检
业务：[名称]
当前 LTV：[数字 / 不知道]
当前 CAC：[数字 / 不知道]
当前复购率：[数字 / 不知道]
```

**默认召唤**：交付可行性专家 + RS + CST + KA

**输出**：单位经济模型 + 现金流健康度评级 + 飞轮图

---

### 6️⃣ `/bmc 市场体检`

**触发场景**：进入新市场、选择合作伙伴、评估竞争

**模板**：
```
/bmc 市场体检
业务：[名称]
目标市场：[地域/人群]
合作方：[候选合作方 / 不知道找谁]
```

**默认召唤**：市场可行性专家 + KP + CH

**输出**：四维雷达图 + 关键依赖清单 + 实现路径建议

---

### 7️⃣ `/bmc 画布`

**触发场景**：已经有结论，要可视化输出

**模板**：
```
/bmc 画布 [类型]
内容：[结论描述]
```

**类型选项**：
- `bmc` — 9 宫格画布
- `flywheel` — 增长飞轮图
- `journey` — 客户旅程图
- `coherence` — 模块咬合关系图
- `timeline` — 优化路径时间线

**示例**：
```
/bmc 画布 bmc
内容：AI 简历优化订阅，CS=30 岁互联网求职者，
VP=30 秒改简历，CH=知乎+猎聘，RS=包月订阅，CST=AI算力
```

## 三、快捷命令（10 个高频问题）

| 快捷命令 | 召唤角色 | 解决问题 |
|---------|---------|---------|
| `/bmc 能赚钱吗` | 交付可行性 + RS + CST | 算不算得过来账 |
| `/bmc 有人买吗` | 客户需求 + CS + VP | 客户真实需求 |
| `/bmc 能进吗` | 市场可行性 + KP + CH | 市场时机与生态 |
| `/bmc 渠道换吗` | CH + CR + KP | 渠道 ROI 评估 |
| `/bmc 该涨价吗` | RS + 客户需求 + VP | 涨价可行性 |
| `/bmc 复购怎么提` | CR + 客户需求 + RS | 复购瓶颈 |
| `/bmc CAC 怎么降` | CH + KA + 客户需求 | 获客成本 |
| `/bmc 飞轮` | 交付可行性 + KA + CR | 增长飞轮 |
| `/bmc 咬合度` | 逻辑审查官 | 模块一致性 |
| `/bmc 季报` | 14 角色全员 | 季度战略复盘 |

## 四、上下文相关命令

### `/bmc 对比`
**触发场景**：对比两个商业模式 / 两个版本

**模板**：
```
/bmc 对比
A：[业务 1 描述]
B：[业务 2 描述]
维度：[全面 / 仅财务 / 仅客户 / 仅市场]
```

---

### `/bmc 复盘`
**触发场景**：诊断已经做过，要基于上次诊断看进化

**模板**：
```
/bmc 复盘 [上次诊断日期]
业务：[名称]
本季度变化：[主要变化]
```

---

### `/bmc 行业 [行业名]`
**触发场景**：在某个特定行业下做诊断（注入行业知识）

**模板**：
```
/bmc 行业 [行业名]
任务：[诊断需求]
```

**支持行业关键词**：
- `SaaS` / `电商` / `内容` / `教育` / `医疗` / `金融` / `本地服务` / `硬件` / `出海` / `B2B` / `B2C` / `平台`

## 七、命令组合技

**进阶用法**：多个命令组合，构造复杂任务

### 组合 1 · 拍砖 + 完整诊断
```
/bmc 拍砖 AI 法律咨询工具
（如果拍砖结论是"值得做"）
→ /bmc 完整诊断
业务：AI 法律咨询工具
...
```

### 组合 2 · 多个专项诊断
```
/bmc 客户需求
业务：XX 产品
（输出 JTBD 后）
→ /bmc 专项诊断 VP
业务：XX 产品
任务：基于 JTBD 重写价值主张
```

### 组合 3 · 对比 + 画布
```
/bmc 对比
A：会员制 vs B：单次付费
（输出对比结论后）
→ /bmc 画布 flywheel
内容：会员制飞轮
```
